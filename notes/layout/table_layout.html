Tables were the original grid. They allow you to specify cells in rows and columns. You can span individual cells across columns and/or rows. Also the tables will resize with the browser window to an extent.

The problems with tables for layout are many fold. For one, they require a lot of extra markup in your HTML to make them work. This can be difficult to author and maintain.

They don't allow empty space without the use of non-breaking-spaces &nbsp; or spacer.gifs. Though they do have the ability to align horizontally and vertically very easily.

They also don't allow you to change the order or layout with CSS for different screen sizes. (Note: yes you can change the elements using CSS, but at that point it's no longer a table!)
